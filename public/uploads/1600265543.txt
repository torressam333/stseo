Architecting a Project

1. Identify all of the most fundamental things your app has to do.
2. Code the fundamentals
3. Structure your code with modules
	-Keeps units of code for a project cleanly separated and organized
	-Encapsulate data into privacy and expose other data publicly.
4. What modules do we need in the project? (Look at your tasks).
	-Which has to do with UI and which work with Data?
	-UI MODULE & DATA MODULE
		.Categorize your tasks according to what needs to be done
